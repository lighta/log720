Cet exemple illustre l'utilisation du framework d'applications web Spring v2.5. Il s'agit d'une version modifiée du tutoriel Developing a Spring Framework MVC application step-by-step. Le tutoriel original a été adapté pour utiliser PostgreSQL et un POM maven a été ajoutéet. De plus, plusieurs corrections ont été apportées, en plus de quelques ajouts, dont l'internationalisation ("i18n", support multilingue).

NOTE IMPORTANTE: en plus de bien expliquer Spring, ce tutoriel met en application plusieurs bonnes pratiques de développement d'applications web, dont entre autres le développement continu de tests au fur et à mesure que de nouvelles fonctionnalités sont ajoutées.

Pré-requis

L'exemple tel que livré ici est autonome. Pour que cette application fonctionne:

  • un compilateur java doit être sur le chemin d'accès ("path");
  • maven doit être sur le chemin d'accès ("path");
  • une base de données PostgreSQL doit exister, être accessible et doit contenir une table "products". Des scripts SQL sont inclus pour créer et peupler cette table dans le répertoire src/main/sql.

Pour exécuter l'exemple à partir d'une console DOS

  • ajuster les paramètres dans le fichier src/main/resources/jdbc.properties afin que la configuration reflète votre contexte;
  • s'assurer que le serveur de base de données est activé, contient une table products qui contient des produits (voir src/main/sql);
  • se déplacer à l'endroit où se trouve le fichier pom.xml dans cette archive (en principe au même niveau que les présentes directives);
  • lancer mvn compile jetty:run
  • pointer un fureteur Web sur http://localhost:8080/spring-mvc

Pour expérimenter avec l'internationalisation (i18n)

Cette version de l'application utilise des "message bundles". Tout le texte affiché dans les différentes pages est centralisé à un seul endroit, de telle sorte que la langue des messages est facile à changer. Deux langues sont supportées dans cette version, l'anglais et le français (tous lieux confondus). Comme la langue d'affichage dépend des réglages de votre fureteur, je vous explique ici comment changer ces réglages pour pouvoir voir l'i18n à l'oeuvre.

  • Dans Google Chrome: Tools → Options, onglet "Under the Hood", Section "Web content", bouton "Language and spell checker settings...", onglet "Languages"
  • Dans Firefox 3+: Tools → Options → Content, section "Languages", bouton "Choose..."
  • Dans Internet Explorer 7+: Tools → Internet Options, onglet "General", bouton "Languages"

Dans tous les cas, ajouter les langues désirées, et les placer en ordre de préférence. Pour fins d'experimentation, ajouter au moins une version de l'anglais et une version de français (n'importe quel pays), changer l'ordre et rafraichir votre fureteur ("reload"). Vous devriez voir l'impact tout de suite.
